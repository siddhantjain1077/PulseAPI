# Contributing to PulseAPI

Thank you for considering contributing! Please follow these guidelines to make the process smooth.

- Open an issue to discuss large changes before starting work.
- Fork the repository and create a feature branch: `git checkout -b feature/your-feature`.
- Keep changes focused and small; open one pull request per logical change.
- Write clear commit messages and include any relevant screenshots or logs.
- Run linters and tests before submitting PRs.

## Development

- Backend:
  - Install: `cd backend && npm install`
  - Run: `npm run dev`

- Frontend:
  - Install: `cd frontend && npm install`
  - Run: `npm run dev`

## Code style

Follow the existing project style. Use ESLint/Prettier if configured.

Thanks — maintainers will review PRs and provide feedback.
